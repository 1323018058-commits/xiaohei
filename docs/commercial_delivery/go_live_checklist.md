# 首批客户上线检查清单

## 目标

在客户正式开始操作真实 Takealot 店铺之前，完成最后一轮上线前检查。

## A. 平台侧准备

- [ ] 最新 `npm run release:preflight` 为绿色，或仅剩已接受的公网域名类跳过项
- [ ] 最新 `npm run host:check` 为绿色，或仅剩已接受的公网域名告警
- [ ] 最新 `npm run ops:data:check` 为绿色
- [ ] Worker 心跳正常
- [ ] 备份与恢复校验足够新

## B. 租户侧准备

- [ ] 租户已按最终名称和 slug 创建
- [ ] 首个租户管理员可登录
- [ ] 套餐已确认
- [ ] `trial_ends_at` 或 `current_period_ends_at` 已设置正确
- [ ] Dashboard 显示正确的订阅状态
- [ ] 租户当前为 `active`

## C. Takealot 接入准备

- [ ] 已确认正确的生产店铺
- [ ] API Key 已通过安全通道收到
- [ ] API Secret 已通过安全通道收到
- [ ] 凭证已在 Store 配置中校验通过
- [ ] 至少完成一次成功或可接受的首轮同步
- [ ] 回流 Listing 确认属于正确店铺

## D. 客户交付准备

- [ ] 客户交付模板已填写
- [ ] 临时密码已通过安全方式发送
- [ ] 客户已知晓首次登录后需要改密码
- [ ] 已提供支持通道
- [ ] 已提供升级联系人

## E. 首日观察

- [ ] 已确认客户首次登录成功
- [ ] 已确认客户首次 Dashboard 读取成功
- [ ] 已确认客户首次店铺读取或同步成功
- [ ] 没有未处理的鉴权失败
- [ ] 没有未处理的异常 `402`
- [ ] 没有未处理且影响客户的任务失败

## 一票否决项

只要以下任一项成立，就不能宣布上线完成：
- 客户不能登录
- Dashboard 状态错误
- 凭证尚未校验
- 首轮同步未复核
- 账号或密钥通过不安全通道交付
