# Takealot API 凭证接入指引

## 目标

安全地收集客户的 Takealot API 凭证，并在小黑 ERP 内完成校验，不泄露 Secret。

## 需要客户提供的内容

- 店铺显示名
- API Key
- API Secret
- 生成该凭证的账号邮箱或账号标识
- 确认该凭证对应生产用 Takealot 店铺

## 客户应该通过什么方式提供

允许：
- 密码管理器安全分享
- 加密凭证库
- 审批过的一对一安全消息

不允许：
- 群聊截图
- 大群明文粘贴
- 不受控共享文档

## 可以发给客户的话术

> 请从您当前使用的 Takealot 卖家/开发环境中生成一组最新的 API Key 和 API Secret，并通过约定的安全通道发给我们。我们会用它在小黑 ERP 中建立店铺连接，并执行一次受控的首轮同步校验。

这份指引故意不绑定某张平台截图或某个按钮路径，避免后续 Takealot 门户调整后文档失效。

## 运营侧校验步骤

拿到凭证后：

1. 打开 `Stores`
2. 新建或更新客户店铺
3. 粘贴 API Key 和 Secret
4. 保存
5. 触发一次同步
6. 确认任务完成和 Listing 回流

## 结果判定

### 正常

- 店铺创建成功
- 同步任务到达 `succeeded` 或可接受的 `partial`
- 返回了正确 Listing

### 凭证无效

典型表现：
- 保存后同步返回鉴权失败
- 或校验阶段直接失败

运营动作：
- 不要盲目重试
- 先确认客户提供的是否是旧 Key、错误环境 Key、或已撤销 Key
- 要求客户重新旋转一组新的生产凭证

### 可能接错店铺

典型表现：
- 同步成功，但回来的 Listing 明显属于其他店铺

运营动作：
- 立即停用该店铺
- 与客户确认凭证来源
- 更换凭证后重新同步

## 什么时候必须旋转凭证

- 客户怀疑 Secret 泄露
- Secret 发错了对象
- 连续鉴权失败且怀疑上游已撤销
- 人员变动需要做隔离

## 最低验收标准

在以下条件未同时满足前，不得把客户标记为“已正式开通”：
- 凭证已在小黑 ERP 中完成校验
- 至少完成过一轮受控同步
- 返回的数据确认属于正确的 Takealot 店铺
