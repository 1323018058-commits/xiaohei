# 首批商用故障响应手册

## 目的

给运营和交付人员一套可以直接复用的动作与对外话术，避免首批客户阶段遇到问题时现场乱改口径。

## 1. 客户无法登录

### 先查什么

- 租户状态：`active / suspended / disabled`
- 用户状态
- 是否刚执行过密码重置
- 最近审计日志

### 运营动作

- 如果租户被暂停或停用，先确认是否允许恢复
- 如果客户忘记密码，走租户管理员密码重置流程
- 如果订阅正常但仍登录失败，用受控账号做一次验证

### 对客户的话术

> 我们正在确认当前是工作区被暂停，还是管理员账号需要重新发放密码。如有需要，我们会立即重新下发管理员密码并确认可登录状态。

## 2. Dashboard 显示 `past_due`

### 这意味着什么

- 试用期或付费有效期已过
- 系统故意暂停写操作

### 运营动作

- 检查 `current_period_ends_at` 或 `trial_ends_at`
- 如果已经确认续费，就立即更新订阅周期
- 更新后确认 `/api/auth/me` 和 Dashboard 已恢复

### 对客户的话术

> 您当前的工作区仍可读取数据，但由于当前订阅周期已结束，写操作被暂停。我们正在补齐续费周期信息并恢复可写状态。

## 3. 创建店铺或同步时返回 `402`

### 运营动作

- 不要先去排查 Takealot
- 先检查租户订阅状态和到期时间
- 商务确认通过后，先恢复付费周期

### 对客户的话术

> 当前不是平台接口故障，而是工作区处于待续费状态，系统因此阻止了写操作。我们正在处理续费状态更新。

## 4. Takealot 同步鉴权失败

### 运营动作

- 与客户确认凭证来源
- 必要时让客户旋转新的 Key / Secret
- 仅做单次受控同步复测

### 对客户的话术

> 当前这组 Takealot 凭证没有通过鉴权。请通过安全通道发送一组新的生产 API Key 和 API Secret，我们会立即重新校验。

## 5. 同步成功但没有 Listing

### 运营动作

- 确认是否接到了正确店铺
- 查看任务事件日志
- 确认上游店铺当前是否确实有可同步商品
- 检查是否被 SKU 搜索或筛选条件隐藏

### 对客户的话术

> 当前连接已建立，但我们还需要确认这个账号下是否存在可导出的商品，以及是否接入了预期的店铺。

## 6. 需要紧急停用租户

适用情况：
- 怀疑账号泄露
- 客户要求立刻停用
- 怀疑接错店铺或数据风险

### 运营动作

- 暂停或停用租户
- 撤销现有会话
- 在审计中写清原因
- 完成隔离后再通知客户

### 对客户的话术

> 我们已先暂停工作区以控制风险，现有登录会话已被撤销。接下来我们会和您确认恢复条件与后续步骤。
