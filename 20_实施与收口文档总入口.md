# 实施与收口文档总入口

## 1. 目的

本文件作为 ERP 重构项目当前阶段的统一实施与收口入口，用于把主 PRD 第 15 章之外继续下沉形成的控制文档、批次文档、冻结文档与收口文档统一收拢，避免团队在 `09~21` 间自行猜入口。

原则：

- 主 PRD 第 15 章继续保留 `04~08` 作为实施准备主索引
- 本文件负责承接 `09~21` 的控制与执行下沉文档
- 根目录 `00~23` 是项目主文档层；`docs/superpowers/plans/` 仅作 agent 镜像草稿，不作为项目主输入
- 后续若 `04~08` 与 `09~19` 出现冲突，优先以对应主版本和控制文档收口结论为准

## 3. 唯一入口规则

### 3.1 若你在做……请优先看

- 阶段判断 / 优先级 / 风险：`00_项目上下文.md`
- 当前任务推进顺序：`01_任务看板.md`
- 为什么这么做：`02_决策日志.md`
- `P0` 接口实现：`10_P0接口实现批次拆解.md`
- `P1` 接口实现：`16_P1接口实现批次拆解.md`
- 数据库迁移批次：`11_数据库迁移批次拆解.md`
- 表级 DDL：`15_Batch1-4表级DDL落地清单.md`
- 页面与交互：`12_前端页面实施清单.md` + `17_页面开发顺序与交互冻结点.md`
- 压测：`13_压测脚本与造数方案.md` + `18_压测执行口径冻结.md`
- 上线：`14_上线彩排与责任矩阵.md`
- 一致性复查：`19_一致性复查与收口清单.md`
- 异常处理 / 人工介入 / 值班：`21_异常处理与人工介入手册.md`

### 3.2 不要怎么用

- 不要把 `09~19` 当成平级散文档随机打开
- 不要绕过 `00 / 01 / 02` 直接扩写执行文档
- 不要在 `04~08` 之外重复发明第二套实施口径
- 不要把 `docs/superpowers/plans/` 里的镜像计划稿误当成正式交付文档

## 4. 与主 PRD 第 15 章的关系

- 主 PRD 第 15 章继续作为“实施准备文档索引”入口，指向 `04~08`
- 本文件作为“实施与收口文档总入口”，承接 `09~19`
- 两者关系是：

```text
主 PRD 第 15 章
  → 04~08（实施准备主文档）
  → 20_实施与收口文档总入口
        → 00/01/02/09（控制文档）
        → 10~19（下沉执行与收口文档）
        → 21（运行与值班手册）
```
