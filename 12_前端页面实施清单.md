# 前端页面实施清单

## 1. 目标

本文件把 `06_页面级线框稿.md` 继续下沉为可执行的前端实施清单，覆盖：

- 页面壳
- 任务流
- 抽屉 / Dialog / 弹层
- 表格 / 卡片 / 图表组件
- 依赖接口
- 验收口径

本清单采用“页面壳 + 任务流 + 通用交互件”的结构，而不是单纯按页面名平铺。

---

## 2. 实施总原则

- 先落 `P0` 页面，再补 `P1`
- 优先落任务闭环，不优先堆展示细节
- 通用件优先抽出，避免页面各自发明
- 高危动作、长任务反馈、状态表达优先级高于视觉装饰

## 7. Fulfillment / PO 任务流

### 7.1 冻结交互决策

本域采用两阶段双 Tab 结构，不使用单页混合工作台：

1. `订单池 Tab`：只负责挑货、筛选、勾选、合单
2. `PO 工作台 Tab`：只负责进入 PO 处理与沉浸式打单

### 7.2 页面

- `Fulfillment 页面`
  - `订单池 Tab`
  - `PO 工作台 Tab`

### 7.3 订单池 Tab

目标：

- 专注“挑货合单”

组件：

- `OrderPoolSummaryCards`
- `OrderPoolFilterBar`
- `OrderPoolTable`
- `MergeToPoActionBar`

交互：

- 操作员通过筛选条件找出同仓订单
- 勾选后点击“合并为 PO”
- 如果混选了不同仓库订单：
  - 合并按钮直接置灰
  - 弹出黑白 `Toast`：`禁止跨仓合单`

依赖接口：

- `GET /api/fulfillment/orders`
- `GET /api/fulfillment/orders/{id}`
- `POST /api/fulfillment/pos`

### 7.4 PO 工作台 Tab

目标：

- 专注“沉浸打单”

组件：

- `PoCardList`
- `PoSummaryCard`
- `PoStatusTag`
- `PoProgressBadge`

交互：

- 页面展示卡片式 `PO` 列表
- 点击某个 `PO` 时，不打开右抽屉
- 而是在屏幕中央打开一个占据约 80% 宽度的毛玻璃 `Dialog`

### 7.5 沉浸式打单 Dialog

组件：

- `PoTicketingDialog`
- `PoItemImageCell(48x48)`
- `PoItemSkuText`
- `TrackingInputGrid`
- `SaveAndCloseFooter`

冻结交互：

- 员工视线里只保留：
  - `48x48` 商品图
  - 粗体 `SKU`
  - 单号输入框
- 输入框支持回车自动跳到下一行
- 没有侧栏、次要菜单和额外干扰
- 打完后点击 `保存并关闭`
- Dialog 消失，返回 `PO 工作台 Tab`

依赖接口：

- `GET /api/fulfillment/pos`
- `GET /api/fulfillment/pos/{id}`
- `POST /api/fulfillment/pos/{id}/tracking`
- `POST /api/fulfillment/pos/{id}/items`

### 7.6 异常处理弹层

- `异常标记弹层`
- 展示：对象、当前状态、异常类型、原因、是否进入人工处理

依赖接口：

- `POST /api/fulfillment/orders/{id}/exception`
- `POST /api/shipments/{id}`
- `POST /api/fulfillment/pos/{id}/cancel`

### 7.7 验收口径

- 禁止跨仓合单必须前端可见、后端强校验
- `订单池 Tab` 与 `PO 工作台 Tab` 职责绝不混淆
- 打单 Dialog 必须沉浸、简洁、可快速连续录入

## 13. 关键冻结点

- `Fulfillment` 采用双 Tab：`订单池` + `PO 工作台`
- `PO` 打单使用居中 80% 宽毛玻璃 Dialog，而非右抽屉
- 跨仓合单前端必须置灰并 Toast，后端必须强校验
- 高危动作统一使用标准确认弹层
