# 附录 A：权限矩阵

## 1. 角色范围

| 角色 | 权限范围 | 定位 |
|---|---|---|
| `super_admin` | 全平台 | 平台所有者，拥有用户、订阅、功能、系统与审计最高管理权 |
| `tenant_admin` | 当前租户 | 客户侧管理员，只能管理本租户用户、店铺与功能 |
| `operator` | 当前租户 / 授权店铺 | 业务运营人员，负责选品、铺货、竞价观察与履约跟进 |
| `warehouse` | 当前租户 / 仓库任务 | 仓库执行人员，负责到仓、扫码、打包、贴标、出库 |
| `system_worker` | 系统内部 | 后台任务执行身份，不允许登录前端 |

## 2. 权限标记

| 标记 | 含义 |
|---|---|
| `R` | 可读 |
| `W` | 可写 |
| `A` | 可审批 / 可管理 |
| `H` | 高危操作，必须二次确认与审计 |
| `RO` | 只读 |
| `-` | 无权限 |

## 4. 数据范围规则

- `super_admin` 可跨租户查看与管理，但所有高危操作必须审计。
- `tenant_admin` 只能访问本租户数据。
- `operator` 只能访问被授权店铺与功能。
- `warehouse` 默认只访问仓库与履约执行相关数据。
- `system_worker` 只能通过任务系统执行，不允许绕过领域服务直接写核心表。

## 5. 功能开关规则

每个用户或租户必须支持以下功能开关：

- `selection`
- `listing`
- `bidding`
- `fulfillment`
- `finance`
- `warehouse`
- `admin`
- `extension`

关闭规则：

- 功能关闭后，历史数据默认只读。
- 功能关闭后，前端应显示禁用原因。
- 功能关闭不能只在前端隐藏，后端接口必须拒绝写操作。

## 6. 高危操作审计字段

所有 `H` 操作必须记录：

- `audit_id`
- `actor_user_id`
- `actor_role`
- `target_type`
- `target_id`
- `action`
- `before`
- `after`
- `reason`
- `request_id`
- `ip`
- `user_agent`
- `created_at`

## 7. 密钥与凭证规则

- 店铺 API Key、外部平台 Token 不允许前端读取明文。
- 前端只展示脱敏值，例如 `****abcd`。
- 凭证更新必须记录审计日志。
- 凭证校验必须通过异步任务执行，不能阻塞主请求。
