# 页面级线框稿

## 1. 目标

本文件把 `03_前端风格与交互规范.md` 与首发业务范围，展开为页面级信息架构和文字线框，供前端设计、交互评审和 API 联调使用。

原则：

- 任务导向，不按数据库表导航
- 页面默认只展示 5~6 个核心字段
- 详情、审计、任务日志优先放右侧抽屉
- 高危动作必须二次确认
- 长任务必须展示 `status / stage / progress / recent events`

---

## 2. 顶层导航

首发一级导航：

- `Dashboard`
- `Selection`

二级或抽屉入口：

## 4. Dashboard

### 4.1 页面目标

让用户在 10 秒内看到：

- 今天最需要处理的任务
- 风险最高的问题
- 店铺与系统是否健康
- 长任务是否卡住

### 4.2 文字线框

```text
[页面标题] Dashboard
[副标题] 今日运营摘要与异常总览

[第一行卡片]
  店铺健康卡 | 待处理订单卡 | 竞价风险卡 | 账号到期提醒卡

[第二行卡片]
  运行中任务卡 | 死信/人工介入卡 | 财务快照更新时间卡 | 外部接口健康卡

[第三行]
  左：今日待办列表（按优先级）
  右：最近异常与高危操作摘要
```

### 4.3 关键交互

- 点击任务卡打开 `Task Drawer`
- 点击异常卡跳转对应模块并带筛选条件
- 顶部细进度条展示当前最重要长任务

## 7. Bidding

### 7.1 自动竞价工作台

```text
[标题] Bidding
[顶部卡片]
  全店竞价状态 | 风险护栏 | 今日调价次数 | 失败/跳过数量

[主区左]
  商品列表
  默认列：图片 | SKU | 当前价/BuyBox | 底价区间 | 状态 | 下一步动作

[主区右抽屉]
  单品详情
  折线图（BuyBox 波动）
  区间带（floor / ceiling）
  最近调价日志
  配置编辑
```

### 7.2 主操作

- 启动全店竞价
- 停止全店竞价
- 导入底价
- 筛选“无底价 / 被护栏拦截 / 失败”

### 7.3 关键交互规则

- Slider 改底价只更新输入框，不自动保存
- 点击“保存配置”后才提交
- 启停全店竞价必须高危确认

```text

[右侧详情]
```

### 8.3 逐单补号体验

### 8.4 异常标记弹层

显示：

- 订单 / 运单对象
- 当前状态
- 异常类型

## 14. 页面与接口对齐要求

- 所有页面消费后端返回的 `status / stage / ui_meta`
- 默认列表字段控制在 5~6 个核心字段
- 所有高危操作必须带二次确认 + 原因
- 所有详情、审计、日志优先进抽屉
- `P1` 页面必须清楚标注“受限可用”
- `P2` Listing 页面首发只允许展示任务壳、历史记录和状态壳
