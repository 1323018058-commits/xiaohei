# ERP 重构项目上下文

## 1. 项目目标

- 项目名称：小黑 ERP / ProfitLens 重构
- 当前目标：在 `2026-05-01` 前完成可上线的战时全量版
- 当前例外：`Listing / Dropship` 自动铺货允许在 `2026-05-03` ~ `2026-05-05` 补齐闭环
- 总原则：
  - 后端给真相，前端做表达
  - 状态机驱动业务，任务流驱动长耗时动作
  - 安全、权限、审计、幂等、监控前置
  - 优先保证商业化可控、系统稳定、数据一致

## 2. 已冻结的高层决策

- 架构形态：模块化单体（Modular Monolith）
- 技术基线：Next.js + Python Backend + PostgreSQL + Redis
- 外部平台：Takealot、1688、Amazon、CNExpress、DeepSeek
- 核心对象：
  - `Store`
  - `ListingJob`
  - `BidProduct`
  - `FulfillmentOrder`
  - `PO`
  - `Shipment`
  - `FinanceSnapshot`
- 核心骨架：
  - `Task`
  - `Status`
  - `Stage`
  - `ui_meta`
- 前端风格：
  - 极简主义与呼吸感
  - 浅灰/纯白背景，圆角白色卡片
  - Apple 风格蓝色主操作
  - 动态列、高信噪比表格、视觉化对账
  - 看板式铺货流水线与金融质感竞价工作台

## 3. 首发范围

### 3.1 `2026-05-01` 前必须上线

- `Auth / Admin / Subscription`
- `Store Management`
- `Product Intelligence / Selection`
- `AutoBid`
- `Fulfillment / PO Workbench`
- `Finance`
- `Warehouse / CNExpress`
- `Extension` 最小版
- 安全与运行底座

### 3.2 `2026-05-03` ~ `2026-05-05` 补齐

- `Listing / Dropship` 自动铺货闭环

## 4. 当前最重要风险

- 状态机不闭环导致业务对象进入死状态
- 外部平台脏数据污染主库
- 重复提交、重复任务、重复调价、重复入账
- 管理员体系不足以支撑商业化账号管理
- 没有持续检查点会导致上下文断裂与推进失序
