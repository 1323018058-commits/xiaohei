# ERP 重构决策日志

## 2026-04-21

### D001：采用战时全量版 PRD 策略

- 决策：重写 `PRD v2.1（战时全量版）`
- 原因：项目必须在 `2026-05-01` 前上线，不能继续沿用理想态节奏
- 影响：文档需要按战时优先级重新组织，而不是沿用原 Phase 顺序

### D002：自动铺货允许延后几天补齐

- 决策：`Listing / Dropship` 自动铺货闭环允许在 `2026-05-03` ~ `2026-05-05` 完成
- 原因：用户明确允许该模块稍慢几天
- 影响：首发文档需把该模块标记为延期补齐，不得与其他模块混为同一上线门槛

### D003：必须补齐平台级超级管理员

- 决策：新增 `super_admin`
- 原因：现有 `Admin` 角色不足以承载商业化用户管理、账号密码管理、到期控制与功能限制
- 影响：PRD 必须补充账号生命周期、权限矩阵、审计与会话治理

### D004：安全与稳定能力必须前置

- 决策：将权限、限流、幂等、审计、监控、备份、回滚前置到首发 PRD
- 原因：目标是商业化、1000 用户导向、可控可审计
- 影响：原 PRD 的 Phase 7 内容需要前移

### D005：建立持续检查点机制

- 决策：通过线程自动检查点 + 本地控制文档保存上下文和任务进度
- 原因：项目推进节奏快，必须防止上下文丢失
- 影响：每次新一轮工作都必须先读取控制文档，再继续推进

### D006：前端采用商业 SaaS 执行控制台风格（已被 D007 修正）

- 决策：前端采用左侧深色导航、右侧浅色内容区、高密度表格、状态徽标、详情抽屉、任务进度中心的商业 SaaS 控制台风格
- 原因：系统面向高频运营、仓库与管理员，不适合营销化、低密度或强动画风格
- 影响：该方向已被 D007 升级为极简 Apple-like 任务型工作台；仍保留“状态真相、下一步动作、高风险确认、长任务可追踪”的原则

### D007：前端视觉方向升级为极简 Apple-like 任务型工作台

- 决策：前端从传统高密度后台风格升级为极简、留白充足、卡片式、高信噪比、任务导向的 Apple-like 商业工具风格
- 原因：用户明确希望减少传统 ERP 拥挤感，用更清爽、更傻瓜式、更适配平板和移动端的交互承载复杂业务
- 影响：默认表格字段减少到 5 到 6 个核心字段，核心模块采用卡片、抽屉、看板、细进度条、视觉化对账、轻量反馈和智能预填

### D008：PRD v2.1 五个上线附录冻结

- 决策：新增并冻结权限矩阵、状态机总表、错误码表、外部接口重试策略、上线门槛与回滚条件五个附录
- 原因：`2026-05-01` 上线需要明确权限边界、业务状态、失败语义、外部依赖容错和事故回滚策略
- 影响：后续 API、前端和 worker 设计必须遵守五个附录，不得绕过权限、状态机、错误码、重试与回滚约束

### D009：API 写接口统一规则冻结

- 决策：新增并冻结 `附录F_API写接口统一规则.md`
- 原因：写接口是最容易失控的部分，必须统一请求头、幂等、审计、任务化和分域规则
- 影响：后续所有写接口设计都必须声明权限、状态前置条件、幂等键、错误码和返回语义，禁止页面导向的临时写法

### D010：任务系统字段与状态全集冻结

- 决策：新增并冻结 `附录G_任务系统字段与状态全集.md`
- 原因：任务系统是全局运行骨架，必须明确字段全集、状态全集、事件全集、租约、重试和死信规则
- 影响：后续所有长任务、worker、watcher、进度展示和取消逻辑必须遵守附录 G，不得私自新增不可追踪状态

### D011：财务快照和流水重算规则冻结

- 决策：新增并冻结 `附录H_财务快照和流水重算规则.md`
- 原因：财务是最不能“差不多”的域，必须保证流水不可变、快照可重算、调整有痕迹
- 影响：后续财务实现不得覆盖流水，不得静默改历史快照，所有重算必须任务化并版本化

### D012：审计字段与高危动作清单冻结

- 决策：新增并冻结 `附录I_审计字段与高危动作清单.md`
- 原因：商业化系统必须能追责、能回溯、能发现异常操作，尤其是账号、权限、竞价、PO、财务和任务恢复
- 影响：后续所有高危动作必须写审计，必须有 request_id、actor、target、before、after、reason、result，不允许审计缺失的静默成功

### D013：PRD v2.1 核心冻结，进入实施准备阶段

- 决策：将当前阶段从“PRD 正文第一版”切换为“PRD 核心冻结完成，进入实施准备”
- 原因：正文、前端规范以及附录 A~I 已基本成型，继续停留在“需求补强”会拖慢落地
- 影响：下一阶段应优先输出 API 契约逐接口展开、Schema Blueprint、页面线框稿、压测计划和上线 Runbook

### D014：实施文档包首版展开完成

- 决策：新增并冻结 `04_API契约逐接口展开.md`、`05_Schema_Blueprint.md`、`06_页面级线框稿.md`、`07_压测计划.md`、`08_上线_Runbook.md` 作为实施准备阶段首版文档包
- 原因：PRD 与附录已冻结，如果没有接口、表结构、页面、压测与上线文档，团队仍无法高效进入实施
