# 上线 Runbook

## 1. 目标

本 Runbook 用于指导 `2026-05-01` 首发窗口的发布、验证、降级和回滚。目标不是描述理想流程，而是给出上线当天可执行的操作顺序和事故处理边界。

---

## 2. 发布角色

| 角色 | 职责 |
|---|---|
| 发布负责人 | 控制发布节奏、宣布开关切换 |
| 后端负责人 | 后端部署、任务系统、回滚 |
| 前端负责人 | 前端部署、页面 smoke |
| 数据负责人 | 备份、迁移、数据核验 |
| 运维负责人 | 监控、告警、网络、限流 |
| 业务验收人 | 核心主链验收 |

---

## 3. 发布前硬门槛

必须全部满足，否则阻断上线：

- 登录、登出、`/api/auth/me` 稳定
- `super_admin` 可进入管理后台并可管理用户、密码、到期、功能权限
- 后端 RBAC 生效
- 高危动作有审计日志
- 核心写接口具备幂等保护
- 长任务有状态、重试与失败原因
- 店铺同步可成功 / 可失败回写
- AutoBid 无底价禁止竞价，调价有日志
- Fulfillment 主链跑通
- 利润快照可读，流水不可覆盖
- 发布前已完成数据库备份
- 有明确应用回滚与数据修正策略
- 核心接口、任务、DB、队列可观测
- 登录限流、API 限流、错误脱敏启用

---

## 4. 发布前检查清单

### 4.1 数据

- 完成数据库全量备份
- 确认迁移脚本可重复执行或可识别已执行
- 确认关键表索引存在
- 确认财务流水不可覆盖

### 4.2 安全

- 开启登录限流
- 开启 API 限流
- 开启错误脱敏
- 确认凭证不返回前端明文
- 确认管理员高危操作弹层与原因输入可用

### 4.3 任务

- 验证任务创建幂等
- 验证任务失败可重试
- 验证死信任务可查看
- 验证取消任务不会留下脏状态

### 4.4 外部接口

- 验证 Takealot 凭证失败
- 验证 Takealot 超时
- 验证 CNExpress 超时
- 验证 1688 / Amazon / AI 失败不会影响 `P0` 主链

### 4.5 前端

- 验证禁用功能显示原因
- 验证长任务不是纯 spinner
- 验证高危动作确认弹层
- 验证错误反馈可读

---

## 5. 发布前版本控制要求

- 当前目录必须已初始化 Git 仓库
- 发布前创建基线提交或发布候选提交
- 关键节点打 tag：
  - `baseline-docs`
  - `rc-2026-05-01`
  - `post-launch-hotfix-*`
- 每次上线变更必须对应可回退提交，不允许一大坨未分段改动

---

## 6. 发布顺序

```text
冻结代码与配置
  → 数据库备份
  → 执行数据库迁移
  → 部署后端 API
  → 部署 worker
  → 部署前端
  → smoke test
  → 开启 super_admin
  → 开启 Auth / Store / Fulfillment
  → 开启 AutoBid 只读
  → 小范围开启 AutoBid 写能力
  → 观察监控
  → 逐步扩大开放
```

---

## 7. 开关节奏

### 7.1 第一阶段：核心可登录

开启：

- `Auth`
- `Admin`
- `Subscription`
- `Task Center`
- `Audit`

验证：

- `super_admin` 登录
- 用户列表可读
- 修改功能开关可写

### 7.2 第二阶段：业务主链

开启：

- `Store Management`
- `Fulfillment`
- `Warehouse` 必要读写

验证：

- 店铺同步任务可创建
- 订单池可读
- PO 可创建与补号

### 7.3 第三阶段：谨慎开放竞价

开启：

- 先开放 `Bidding` 查看
- 再按店铺白名单开放 `AutoBid` 写能力

验证：

- 无底价商品不能启用
- 调价日志落库
- 审计日志完整

### 7.4 第四阶段：受限能力

开启：

- `Finance` 只读或受限重算
- `Selection`
- `Extension`

### 7.5 延期能力

- `Listing / Dropship` 仅保留任务入口、历史记录、状态壳
- 完整闭环在 `2026-05-03 ~ 2026-05-05` 补齐

---

## 8. Smoke Test

### 8.1 Auth / Admin

- 登录成功
- `/api/auth/me` 返回正确角色与功能开关
- `super_admin` 可查看用户列表
- 修改单个用户功能权限成功并写审计

### 8.2 Store

- 店铺列表可读
- 触发一次店铺同步，返回 `task_id`
- 任务中心可看到同步任务状态变化

### 8.3 Fulfillment

- 订单池列表可读
- 创建一个 PO 成功
- 逐单补号成功
- 审计可回查

### 8.4 Bidding

- 商品列表可读
- 无底价商品无法启用竞价
- 启动单个测试店铺的竞价任务成功

### 8.5 Finance

- 快照列表可读
- 触发一次重算任务成功返回 `task_id`

## 9. 监控观察窗口

上线后前 24 小时重点观察：

- 登录失败率
- `/api/auth/me` 错误率
- 店铺同步成功率
- 任务死信数量
- AutoBid 调价数量与失败率
- PO 创建与绑定异常
- 外部 API 超时数量
- 数据库连接池使用率
- 队列积压
- 管理员高危操作审计完整性

---

## 10. 降级触发条件

满足任一条件必须进入降级决策：

- 登录大面积失败
- `/api/auth/me` 明显不稳定
- `super_admin` 无法进入后台
- 外部平台异常拖垮主站
- 队列积压持续增长
- AutoBid 调价失败率异常
- PO 主链异常率升高
- 监控与日志不可用

---

## 11. 降级顺序

1. 关闭 `Listing / Dropship` 自动任务
2. 暂停大规模抓取
3. 暂停全店自动竞价，只保留查看
4. 暂停外部同步，只保留核心读能力
5. 管理后台保留 `super_admin` 登录与应急开关
6. 必要时进入整站维护模式

---

## 12. 回滚触发条件

满足任一条件必须进入回滚或数据修正决策：

- 重复调价
- 重复财务入账
- PO 绑定关系错乱
- 数据库迁移导致核心表不可读写
- 5xx 持续升高
- 队列无法消费且恢复失败

---

## 13. 回滚策略

### 13.1 应用回滚

- 优先回滚到最近稳定 tag 或提交
- 回滚前先暂停 worker
- 回滚 API 和前端到同一兼容版本
- 回滚后立即执行 smoke test

### 13.2 数据修正

- 财务错误不做物理回滚，使用 adjustment 修正
- 调价错误优先通过反向调价修正
- PO 错绑通过受审计人工修正
- 整库恢复只作为最后手段

### 13.3 Worker 回滚

- 暂停 worker
- 停止新任务入队
- 保留队列和 dead letter
- 修复后从安全 checkpoint 恢复

---

## 14. Git 级回退约定

### 14.1 文档版回退

适用于当前阶段的控制文档与实施文档：

- 查看提交：`git log --oneline --decorate`
- 回看内容：`git show <commit>`
- 若需回到某一稳定文档点，优先新建恢复提交，不直接破坏历史

### 14.2 发布版回退

适用于后续代码上线阶段：

- 优先回退到发布 tag
- 不允许未核对就 `reset --hard`
- 回滚后必须重做 smoke test 与关键写路径验证

---

- 每个阶段切换前由发布负责人明确口头确认
- 每次降级或回滚必须记录时间、原因、影响范围、决策人
- 发布结束后 24 小时内沉淀问题列表与后续修复计划
