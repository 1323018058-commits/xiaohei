# 上线彩排与责任矩阵

## 1. 目标

本文件把 `08_上线_Runbook.md` 继续下沉为上线当天可直接执行的细化手册，覆盖：

- 责任矩阵
- 发布冻结清单
- 彩排剧本
- 开关表
- Smoke 验收表
- 降级决策矩阵
- 回滚口令与操作表
- 值守与沟通节奏

---

## 2. 发布治理总原则

- 发布负责人统一口头确认节奏
- `super_admin` 后台预置应急开关
- 各域负责人可执行本域操作，但必须在发布负责人确认后进行
- 所有降级与回滚动作必须记录时间、原因、影响范围、执行人与确认人

---

## 3. 角色与责任矩阵

| 角色 | 上线前 | 上线中 | 异常时 | 上线后 24h |
|---|---|---|---|---|
| 发布负责人 | 审核清单、确认窗口、宣布冻结 | 统一确认每个阶段切换 | 决定降级/回滚是否触发 | 组织复盘 |
| 后端负责人 | 确认 API / Worker / 配置就绪 | 部署后端与 Worker | 执行 API / Worker 回滚 | 跟踪 5xx、任务异常 |
| 前端负责人 | 确认页面 smoke 路径 | 部署前端、验证主页面 | 执行前端回退 | 跟踪页面错误与可用性 |
| 数据负责人 | 备份、迁移、索引核查 | 执行迁移与数据核验 | 决定数据修正策略 | 跟踪数据一致性 |
| 运维负责人 | 监控、限流、告警就绪 | 观察基础设施、切流 | 执行限流/维护模式/网络策略 | 跟踪容量与告警 |
| 业务验收人 | 熟悉验收清单 | 验证主链是否可用 | 判断业务侧是否可接受降级 | 收集业务反馈 |
| `super_admin` 操作人 | 准备后台应急开关账号 | 按发布负责人确认执行开关 | 执行降级开关与紧急封禁 | 跟踪高危后台操作 |

## 5. 彩排剧本（按时间线）

### T-120 分钟

- 发布负责人召开上线准备确认
- 数据负责人确认备份策略与恢复点
- 运维负责人确认监控与告警全部在线
- 前后端负责人确认待发布版本号

### T-90 分钟

- 执行数据库备份
- 核对迁移脚本清单
- 核对回滚脚本 / 回滚步骤可用

### T-60 分钟

- 后端负责人部署预发布版本到上线环境待切换位
- Worker 待命但不放量
- 前端待发布版本已可切换

### T-30 分钟

- 发布负责人宣布进入冻结
- 停止无关变更与人工修数
- `super_admin` 操作人登录后台待命

### T-10 分钟

- 数据负责人执行迁移
- 后端负责人确认 API 健康
- 运维负责人确认无异常告警放大

### T+0

- 切换后端
- 切换 Worker
- 切换前端
- 发布负责人宣布进入 Smoke 阶段

### T+10 分钟

- 业务验收人执行 `Auth / Admin / Store / Fulfillment / Bidding / Finance` smoke
- 运维负责人同步观察关键指标

### T+20 分钟

- 开启 `super_admin`
- 开启 `Auth / Store / Fulfillment`
- `AutoBid` 先只读

### T+30 分钟

- 小范围开放 `AutoBid` 写能力
- 观察 10~15 分钟

### T+45 分钟

- 若无严重异常，逐步扩大开放

## 6. 开关表

### 6.1 开关治理规则

- 应急开关预置在 `super_admin` 后台
- 实际执行前，必须由发布负责人明确口头确认
- 执行人、确认人、执行时间必须留痕

### 6.2 开关清单

| 开关 | 默认 | 执行人 | 确认人 | 用途 |
|---|---|---|---|---|
| `auth_enabled` | 开 | `super_admin` 操作人 | 发布负责人 | 登录主开关 |
| `admin_enabled` | 开 | `super_admin` 操作人 | 发布负责人 | Admin 主开关 |
| `store_sync_enabled` | 开 | `super_admin` 操作人 | 发布负责人 | 店铺同步开关 |
| `fulfillment_write_enabled` | 开 | `super_admin` 操作人 | 发布负责人 | Fulfillment 写能力 |
| `autobid_read_enabled` | 开 | `super_admin` 操作人 | 发布负责人 | 竞价查看能力 |
| `autobid_write_enabled` | 关 | `super_admin` 操作人 | 发布负责人 | 竞价写能力 |
| `listing_jobs_enabled` | 关 | `super_admin` 操作人 | 发布负责人 | 自动铺货任务 |
| `finance_recalc_enabled` | 受限 | `super_admin` 操作人 | 发布负责人 | 财务重算 |
| `maintenance_mode` | 关 | 运维 / `super_admin` | 发布负责人 | 整站维护模式 |

## 9. 降级决策矩阵

| 触发条件 | 首选动作 | 确认人 | 执行人 |
|---|---|---|---|
| 登录异常放大 | 保留 `auth_enabled`，先限流再查因 | 发布负责人 | 运维负责人 |
| `super_admin` 无法进入后台 | 阻断继续开放，优先恢复 Admin | 发布负责人 | 后端负责人 |
| 店铺同步拖垮主站 | 关闭 `store_sync_enabled` | 发布负责人 | `super_admin` 操作人 |
| PO 主链异常 | 保留读，关闭 Fulfillment 写能力 | 发布负责人 | `super_admin` 操作人 |
| `AutoBid` 失败率异常 | 关闭 `autobid_write_enabled` | 发布负责人 | `super_admin` 操作人 |
| 外部平台异常拖垮主站 | 关闭相关外部同步与任务入口 | 发布负责人 | 运维 / `super_admin` |
| 监控失效 | 暂停继续放量 | 发布负责人 | 运维负责人 |
| 队列积压失控 | 停止新任务入队 | 发布负责人 | 后端负责人 |

## 9. 回滚口令与操作表

### 9.1 回滚口令

统一口令格式：

- `进入降级一档`
- `进入降级二档`
- `停止继续放量`
- `执行应用回滚`
- `执行 Worker 回滚`
- `进入维护模式`

说明：

- 只有发布负责人可以发出口令
- 其他人不得自行升级为回滚动作

### 9.2 应用回滚操作表

| 动作 | 执行人 | 前置条件 | 后置验证 |
|---|---|---|---|
| API 回滚到上一稳定版本 | 后端负责人 | 已暂停 Worker 新任务 | Auth / Store / Fulfillment smoke |
| 前端回滚到上一稳定版本 | 前端负责人 | API 已确认兼容 | 主页面 smoke |
| Worker 回滚 | 后端负责人 | 已停止新任务入队 | 队列恢复消费 |

### 9.3 数据修正操作表

| 场景 | 动作 | 执行人 |
|---|---|---|
| 重复财务入账 | adjustment 修正 | 数据负责人 |
| 重复调价 | 反向调价修正 | 后端负责人 |
| PO 错绑 | 受审计人工修正 | 数据负责人 + 业务验收人 |

## 10. 值守与沟通节奏

### 10.1 值守分工

- 发布负责人：全程在线
- 后端负责人：全程在线
- 前端负责人：至少上线后 2 小时在线
- 数据负责人：上线前后关键窗口在线
- 运维负责人：全程在线
- 业务验收人：Smoke 阶段与上线后 2 小时在线
- `super_admin` 操作人：开关阶段全程在线

### 10.2 沟通频率

- 每次阶段切换前：发布负责人确认
- 每 15 分钟：同步一次状态
- 每次异常：立即记录与播报
- 每次降级 / 回滚：必须有明确复述确认

## 12. 关键冻结点

- 发布当天开关由 `super_admin` 后台预置，但必须由发布负责人统一口头确认
- 降级优先于整站回滚
- 财务与审计类数据不做物理回滚
- 所有回滚与降级动作必须留痕
