# First Codebase Structure Draft

## 1. 目标

本文件用于冻结第一批实际开工所需的代码目录结构，避免程序开始后因目录分歧导致返工。

## 2. 总体原则

- Python 只保留一个后端根：`apps/api`
- Worker 与 API 共用同一套领域层
- 前端强制隔离 `(auth)` 与 `(dashboard)`
- 契约由后端 OpenAPI 生成，不做手工双写
- 高危交互与长任务反馈优先沉淀为基建件

## 9. 首批代码切片与目录对应

### Slice 1：Auth

- 后端：`modules/auth`
- 前端：`app/(auth)/login`

### Slice 2：Admin

- 后端：`modules/admin`
- 前端：`app/(dashboard)/admin`
- 共享：`components/risk/DangerousActionDialog.tsx`

### Slice 3：Task

- 后端：`modules/tasking`
- 前端：`app/(dashboard)/tasks`
- 共享：`components/tasking/`

### Slice 4：Store

- 后端：`modules/store`
- 前端：`app/(dashboard)/stores`

## 10. 冻结规则

- 程序开工后，不允许再新增第二个 Python 后端根
- 不允许前端绕过生成链手写 API 类型
- 不允许把登录页放回 Dashboard Layout
- 不允许业务页面各自实现高危动作弹窗
